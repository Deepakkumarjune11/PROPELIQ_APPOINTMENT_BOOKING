# User Story - US_029

## Story ID
* ID: US_029

## Story Title
* AI Prompt Safety & RAG Access Controls

## Description
* As a security engineer, I want prompt injection defenses and row-level access controls on the RAG pipeline, so that malicious inputs cannot manipulate AI outputs and users can only retrieve documents they are authorized to access.

## Acceptance Criteria
1. **Given** a user submits text to any AI-powered feature, **When** the input reaches the AI gateway, **Then** the system sanitizes the input against known prompt injection patterns (instruction override, context escape, role impersonation) per AIR-S01.

2. **Given** a prompt injection attack is detected, **When** the sanitizer flags the input, **Then** the system blocks the request, returns a safe error message, and logs the attempt with full context in the audit log per AIR-S01.

3. **Given** a user queries the RAG pipeline, **When** the vector search retrieves document chunks, **Then** only chunks belonging to documents the user has authorization to access are included in the context window per AIR-S02.

4. **Given** a staff member queries patient records via RAG, **When** the retrieval executes, **Then** only patients assigned to that staff member's department or care team are included in results per AIR-S02.

5. **Given** a new prompt injection pattern is discovered, **When** the pattern is added to the blocklist, **Then** the sanitizer is updated via configuration without requiring redeployment per AIR-S01.

## Edge Cases
* What happens when a legitimate clinical query resembles a prompt injection? (System flags for manual review but does not block; staff can override with audit trail)
* How does the system handle encoded or obfuscated injection attempts? (Multi-layer sanitization including Unicode normalization and encoding detection)
