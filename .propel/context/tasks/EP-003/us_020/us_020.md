# User Story - US_020

## Story ID
* ID: US_020

## Story Title
* RAG Extraction & Fact Persistence

## Description
* As a system, I want to extract structured clinical facts from document chunks using RAG retrieval with GPT-4 Turbo and persist results with confidence scores and source citations, so that staff can review AI-extracted data with full provenance traceability.

## Acceptance Criteria
1. **Given** embeddings exist for a document, **When** the extraction pipeline runs, **Then** the system retrieves top-5 chunks via cosine similarity with a threshold of 0.7+ per AIR-R02.

2. **Given** relevant chunks are retrieved, **When** the RAG context is assembled, **Then** chunks are re-ranked by semantic relevance and the context window is limited to 3,000 tokens per AIR-R03 and AIR-R04.

3. **Given** the context is assembled, **When** GPT-4 Turbo processes the extraction prompt, **Then** structured clinical facts (vitals, medications, history, diagnoses, procedures) are returned with confidence scores and character-level source references per AIR-001 and AIR-006.

4. **Given** extraction produces facts with confidence >= 70%, **When** results are persisted, **Then** ExtractedFact records are created with document_id, fact_type, value, confidence_score, source_char_offset, and source_char_length and extraction_status is set to "completed" per AIR-007.

5. **Given** extraction confidence is below 70%, **When** the threshold check fails, **Then** extraction_status is set to "manual_review" and the document is flagged for staff review without discarding extracted data per AIR-007.

6. **Given** extraction targets 2-minute chart-prep, **When** the full pipeline completes, **Then** processing finishes within 30 seconds at p95 per NFR-002.

## Edge Cases
* What happens when no chunks meet the 0.7 cosine similarity threshold? (System flags document as "low_relevance", sets extraction_status to "manual_review")
* How does the system handle extraction for documents with mixed medical and non-medical content? (RAG retrieval naturally filters relevant chunks; non-medical content scores below threshold and is excluded)
