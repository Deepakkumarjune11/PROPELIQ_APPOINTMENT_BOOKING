# User Story - US_016

## Story ID
* ID: US_016

## Story Title
* Staff Walk-In Booking & Patient Creation

## Description
* As a staff member, I want to book walk-in appointments and optionally create patient accounts during intake, so that I can efficiently onboard walk-in patients while maintaining operational control over same-day scheduling.

## Acceptance Criteria
1. **Given** I am an authenticated staff user, **When** I access the walk-in booking form, **Then** the system verifies my staff permissions and displays the walk-in booking interface per FR-008.

2. **Given** I search for a patient by email or phone, **When** a matching patient is found, **Then** the system displays the existing patient details for walk-in booking association.

3. **Given** no patient account exists, **When** I choose to create a patient account during walk-in, **Then** the system creates a minimal patient profile with the provided details per FR-008.

4. **Given** I submit a walk-in booking, **When** a same-day slot is available, **Then** the system creates an appointment with walk-in status and adds the patient to the same-day queue with a position number.

5. **Given** no same-day slots are available, **When** I attempt a walk-in booking, **Then** the system adds the patient to a wait queue managed by staff and displays the wait position.

## Edge Cases
* What happens when a patient user tries to access walk-in booking? (System enforces role-based access control and rejects with 403 Forbidden; walk-in is staff-only per FR-008)
* How does the system handle duplicate patient creation? (Staff search autocomplete matches existing patients by email/phone before allowing new account creation)
