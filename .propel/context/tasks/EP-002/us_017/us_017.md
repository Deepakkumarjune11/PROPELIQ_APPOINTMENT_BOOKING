# User Story - US_017

## Story ID
* ID: US_017

## Story Title
* Same-Day Queue & Arrival Management

## Description
* As a staff member, I want to manage the same-day patient queue and mark patients as arrived, so that I can maintain real-time visibility into patient flow and ensure orderly same-day operations.

## Acceptance Criteria
1. **Given** I am on the staff dashboard, **When** I view the same-day queue, **Then** all queued patients are displayed with position, name, appointment time, and status (waiting/arrived/in-room/completed).

2. **Given** the queue is displayed, **When** I reorder queue entries by drag-and-drop, **Then** the queue positions update in real-time and the change is persisted with Redis cache invalidation.

3. **Given** a patient is physically present, **When** I click "Mark Arrived", **Then** the appointment status transitions to "arrived", the queue state updates, and an audit log entry is created.

4. **Given** the queue cache is fresh (within 30s TTL), **When** another staff member views the queue, **Then** they see the cached queue state for consistent real-time display per NFR-001.

5. **Given** the system supports horizontal scaling, **When** multiple staff users access the queue simultaneously, **Then** the system maintains response times within 2 seconds at p95 per NFR-008.

## Edge Cases
* What happens when a patient leaves before being seen? (Staff marks status as "left" which removes from active queue and creates audit entry)
* How does the system handle queue updates when Redis cache is invalidated? (Next request fetches fresh data from database, rebuilds cache with 30s TTL)
