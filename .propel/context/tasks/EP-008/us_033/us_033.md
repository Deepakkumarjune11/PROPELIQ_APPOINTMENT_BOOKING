# User Story - US_033

## Story ID
* ID: US_033

## Story Title
* Operational Metrics Dashboard & Reporting

## Description
* As a clinic manager, I want a real-time operational metrics dashboard showing appointment volumes, no-show rates, AI utilization, and system health, so that I can make data-driven decisions about clinic operations and resource allocation.

## Acceptance Criteria
1. **Given** I am authenticated as staff or admin, **When** I navigate to the analytics dashboard (SCR-028), **Then** I see real-time KPI cards for today's appointment count, no-show rate, average wait time, and AI suggestion acceptance rate per FR-018.

2. **Given** the dashboard is loaded, **When** I select a date range filter, **Then** all charts and metrics update to reflect the selected period within 2 seconds per NFR-018.

3. **Given** operational data is available, **When** I view the dashboard, **Then** I see trend charts for daily appointment volumes, weekly no-show rates, document processing throughput, and AI response latencies per FR-018.

4. **Given** system health metrics are collected, **When** I view the system health panel, **Then** I see API response time p50/p95/p99, database connection pool usage, cache hit ratios, and AI gateway status per TR-017.

5. **Given** I need to share metrics, **When** I click "Export", **Then** the system generates a PDF or CSV report of the currently displayed metrics and date range per FR-018.

## Edge Cases
* What happens when metrics collection is delayed? (Dashboard shows last-known values with a "data delayed" indicator and timestamp of last refresh)
* How does the system handle dashboard access during high-load periods? (Metrics are pre-aggregated via materialized views; dashboard queries don't impact operational performance)
