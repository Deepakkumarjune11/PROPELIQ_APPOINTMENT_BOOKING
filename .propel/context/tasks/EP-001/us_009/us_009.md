# User Story - US_009

## Story ID
* ID: US_009

## Story Title
* Appointment Availability Search

## Description
* As a patient, I want to search for available appointment slots by date, so that I can find a convenient time for my visit without staff assistance.

## Acceptance Criteria
1. **Given** I am on the availability search page, **When** I select a date range, **Then** the system displays all available appointment slots within that range within 2 seconds at p95.

2. **Given** availability data is requested, **When** the same date range is queried within 60 seconds, **Then** the system returns results from Redis cache without querying the database.

3. **Given** the availability cache has expired (>60s TTL), **When** a new search is performed, **Then** the system queries the database, updates the cache, and returns fresh results.

4. **Given** no slots are available for the selected dates, **When** the search completes, **Then** the system displays an empty state with a suggestion to try different dates or contact the clinic.

## Edge Cases
* What happens when Redis cache is unavailable? (System falls back to direct database query and logs cache miss event)
* How does the system handle concurrent slot claims? (Optimistic UI shows slot as available, but booking transaction validates slot availability atomically)
