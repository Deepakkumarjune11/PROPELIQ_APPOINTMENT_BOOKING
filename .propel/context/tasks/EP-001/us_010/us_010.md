# User Story - US_010

## Story ID
* ID: US_010

## Story Title
* Patient Registration & Insurance Validation

## Description
* As a patient, I want to register my contact and insurance details during booking and receive soft insurance validation, so that my appointment is created with verified information without blocking the booking on validation failures.

## Acceptance Criteria
1. **Given** I have selected an appointment slot, **When** I submit my email, name, DOB, phone, insurance provider, and member ID, **Then** the system creates a patient record and associates it with the selected appointment.

2. **Given** I provide insurance details, **When** the system validates against the internal dummy reference set, **Then** the result is recorded as pass, partial-match, or fail without blocking appointment creation per FR-009.

3. **Given** insurance validation returns partial-match or fail, **When** the booking is confirmed, **Then** the appointment is marked for staff follow-up and the patient receives confirmation regardless.

4. **Given** a patient with the same email already exists, **When** I attempt to register, **Then** the system associates the new appointment with the existing patient account rather than creating a duplicate.

## Edge Cases
* What happens when the insurance reference data service is unavailable? (Validation is skipped, result recorded as "pending", appointment proceeds normally)
* How does the system handle invalid phone number formats? (Inline validation rejects non-standard formats on blur with format guidance message)
