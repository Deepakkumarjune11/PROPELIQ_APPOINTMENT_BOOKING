# User Story - US_014

## Story ID
* ID: US_014

## Story Title
* Reminders, Calendar Sync & PDF Confirmation

## Description
* As a patient, I want to receive SMS/email reminders, calendar events in Google and Outlook, and a PDF confirmation after booking, so that I have complete scheduling artifacts to reduce no-show risk.

## Acceptance Criteria
1. **Given** an appointment is successfully booked, **When** the booking transaction completes, **Then** the system queues background jobs to send SMS via Twilio and email via SendGrid with appointment details per FR-007.

2. **Given** the patient has booked an appointment, **When** the booking confirmation screen loads, **Then** buttons for "Add to Google Calendar" and "Add to Outlook Calendar" are available and functional via OAuth 2.0 APIs per TR-012.

3. **Given** calendar sync is requested, **When** the patient clicks a calendar button, **Then** a calendar event is created in the selected provider with appointment date, time, location, and visit reason per FR-007.

4. **Given** the booking is confirmed, **When** the PDF generation job runs, **Then** a PDF appointment confirmation is generated using PDFSharp and emailed to the patient with booking details per TR-014.

5. **Given** the booking confirmation screen is displayed, **When** the patient views it, **Then** a PDF download link, calendar sync options, and success message are shown per SCR-006.

## Edge Cases
* What happens when SMS delivery fails? (System logs failure, retries once, and records communication status for staff visibility per FR-007 extension 5a)
* What happens when calendar sync fails? (Appointment is preserved, failure logged for retry, patient sees error toast with "Try again" option)
* How does the system handle Twilio/SendGrid free-tier rate limits? (Queue priority ensures confirmations are sent first; low-priority reminders may be delayed)
