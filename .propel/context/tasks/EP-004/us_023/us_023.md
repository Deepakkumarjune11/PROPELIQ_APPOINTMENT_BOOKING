# User Story - US_023

## Story ID
* ID: US_023

## Story Title
* ICD-10/CPT Code Suggestion & Verification

## Description
* As a staff member, I want to review AI-generated ICD-10 and CPT code suggestions with supporting evidence and confirm or reject each code, so that medical coding is accurate, evidence-based, and achieves >98% AI-human agreement before becoming part of the patient record.

## Acceptance Criteria
1. **Given** the 360-degree view is assembled, **When** code suggestions are generated, **Then** ICD-10 and CPT codes are displayed with supporting evidence breadcrumbs linking to the extracted facts per FR-014 and AIR-005.

2. **Given** a code suggestion is displayed, **When** I click on the evidence breadcrumb, **Then** the system navigates to the specific source fact and original document segment.

3. **Given** I review a code suggestion, **When** I click Accept, **Then** staff_reviewed is set to true, review_outcome is recorded, and an audit log entry captures my confirmation per FR-014.

4. **Given** I review a code suggestion, **When** I click Reject with justification, **Then** the code is marked as rejected with my justification text and the AI-human agreement metrics are updated.

5. **Given** all codes are reviewed and conflicts resolved, **When** I click "Finalize Patient Summary", **Then** verification_status is set to "verified", the version is incremented, and the system displays chart prep completion time per UC-005.

6. **Given** AI-human agreement tracking is active, **When** reviewing agreement rates, **Then** the system maintains a rate greater than 98% for extracted facts and suggested codes per AIR-Q05 and NFR-009.

## Edge Cases
* What happens when staff rejects all suggested codes? (System allows "Reject all" with mandatory justification text field; tracks AI-human agreement degradation per figma_spec edge case)
* How does the system handle hallucinated codes not supported by evidence? (Hallucination rate is monitored; codes without supporting evidence are flagged with low confidence per AIR-Q01/NFR-010)
