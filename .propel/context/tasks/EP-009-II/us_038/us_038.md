# User Story - US_038

## Story ID
* ID: US_038

## Story Title
* Loading Feedback, Toasts & Progress Indicators

## Description
* As a user performing actions on the platform, I want immediate visual feedback through skeleton loaders, progress indicators, and toast notifications, so that I always know the system status and never feel uncertain about whether my action succeeded.

## Acceptance Criteria
1. **Given** a page or component is loading data, **When** the content is not yet available, **Then** skeleton loaders matching the content layout are displayed (not spinners) within 100ms of the request per UXR-401.

2. **Given** I perform a successful action (save, submit, delete), **When** the action completes, **Then** a toast notification appears in the top-right corner with a success message, auto-dismisses after 5 seconds, and is dismissible via close button per UXR-402.

3. **Given** I perform an action that fails, **When** the error occurs, **Then** an error toast appears with a meaningful message (not technical jargon), persists until dismissed, and includes a "Retry" action when applicable per UXR-402.

4. **Given** a long-running operation (file upload, document processing, AI analysis), **When** the operation is in progress, **Then** a progress bar or percentage indicator shows real-time progress with estimated time remaining per UXR-403.

5. **Given** multiple toasts are triggered in quick succession, **When** they queue up, **Then** maximum 3 toasts are visible simultaneously, new toasts stack below existing ones, and oldest toasts auto-dismiss first per UXR-404.

## Edge Cases
* What happens when a progress indicator stalls? (After 30 seconds of no progress update, show "This is taking longer than expected" with a cancel option)
* How does the system handle toast notifications when the page is not visible? (Queue notifications; display all queued toasts when the user returns to the tab)
